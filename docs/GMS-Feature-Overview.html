1. Executive Summary

The Grant Management System (GMS) is a comprehensive, web-based platform designed to manage the complete lifecycle of grants — from pipeline identification through closeout — for organizations managing federal, state, foundation, and corporate awards. The system consolidates financial tracking, compliance management, audit trails, document storage, and client communications into a single, secure platform.

Strategic Opportunity: For Citrin Cooperman, the GMS represents a vertical expansion opportunity — extending the firm's accounting, taxation, auditing, and advisory services into grant management as a dedicated service line, deepening client engagement, generating recurring revenue, and creating competitive differentiation.

3. Role-Based Access & Permissions

Strict four-tier role hierarchy. Every action, page, and data point is governed by role and organizational assignment.

Superadmin

Full Platform Control — Senior IT administrators or designated firm-level administrators
  • User Management — Create, edit, deactivate, force password resets
  • Organization Mgmt — Create/edit client orgs, assign staff
  • System Settings — Security, email, branding, feature toggles
  • Audit Log — View & export system-wide audit trail
  • Activity Monitor — Login history, sessions, failed attempts
  • Force-Logout — Terminate any active session
  • Integration Access Keys — Create, revoke, and manage access for third-party systems
  • Health Dashboard — Platform stats & error summaries
  • Backup & Restore — Manual backups, restore from files
  • Portfolio Views — Aggregated cross-org data
  • Announcements — System-wide with acknowledgment tracking

Manager

Operational Grant Management — Grant managers, senior accountants, engagement leads
  • Org Access — Restricted to assigned organizations only
  • Grant Lifecycle — Create, edit, transition statuses
  • Amendments — Before/after snapshot recording
  • Workbooks — Open, lock, submit, approve
  • Financial Ops — Budget, expenses, drawdowns, invoicing
  • Compliance — Requirements, submissions, documents
  • Single Audit — SEFA tracking, auditor coordination
  • Rates — Indirect cost & fringe benefits
  • Two-Factor Authentication — Verification codes enabled for enhanced login security
  • PPA — Two-person control (no self-approval)
  • Reports — Generate & export Excel/PDF

Staff

  • Workbook Entry — Expenses, adjustments, journals
  • Budget Lines — Add and edit line items
  • Drawdowns — Create requests, record receipts
  • Cannot: Lock workbooks, authorize PPAs, create users, transition lifecycle

12. Security

Security is a foundational design principle. Every layer — authentication through data storage to transmission — is built with enterprise-grade controls.

12.1 Authentication

Email-based login with secure verification, four-tier role-based access control, organization-level isolation, session-bound tokens.

12.2 Progressive Lockout (Cool-Off Periods)

5 min5 attempts · Level 1
15 min10 attempts · Level 2
30 min15 attempts · Level 3
60 min20+ attempts · Max
Dual tracking: By both IP and email simultaneously. All failed attempts permanently logged.

12.3 Two-Factor Authentication

Setup

  • Time-based verification codes
  • QR code wizard + manual entry
  • No external dependencies

Protection

  • 30-sec rotation, ±1 tolerance
  • 8 backup recovery codes
  • Replay prevention, 5-attempt lockout

Compatible with Microsoft Authenticator, Google Authenticator, Authy. Manager & Superadmin roles.

12.4 Session Security

MechanismProtection
ID RegenerationNew session ID on login — prevents fixation
FingerprintingSecure binding of session to device and network — detects hijacking
Absolute Timeout8-hour max session
Idle Timeout30 min (configurable)
Role RevalidationEvery 5 minutes from DB
Force-LogoutImmediate session termination by admin
DB TrackingIP, device, start, last activity

12.5 Password & IP Security

  • Industry-standard password encryption, configurable policy (length, upper, lower, numbers, special)
  • History: prevents reuse of last 5 passwords | Force reset capability
  • Every action tied to IP address and device — complete access trail across all devices
  • Org-level IP allowlisting with network range restrictions | Superadmin exempt | All blocks logged

12.6 Audit Logging

What Is Logged

  • Every create, update, and delete action (with before/after values recorded)
  • All authentication events (login, two-factor verification, etc.)
  • Access events, IP blocks
  • Exports and integration calls
  • Admin actions (user/role changes)

Data Per Event

  • User ID + Org context
  • IP address + device information
  • UTC timestamp
  • Old/new values as structured data
  • Description/reason

Excel export for SOC 2 evidence. 10-year retention. Append-only (immutable).

12.7 Integration, Headers & Data Protection

  • Third-Party Access: Secure access token (encrypted), 100 req/hr rate limit, read-only, all calls logged
  • Headers: Comprehensive browser security headers (prevents unauthorized embedding, script injection, and cross-site attacks)
  • Form Validation: Form validation tokens required on all data submissions (prevents unauthorized form submissions)
  • Data: Secure database queries and output encoding (prevents unauthorized data access and script injection), soft-delete, HTTPS enforced
  • Retention: 7 years grants, 10 years audit logs (configurable)

12.8 Multi-Tenant Isolation & Azure Security

Complete segregation: Every query filtered by org ID. Staff access only assigned orgs. Clients locked to own org — cross-tenant access architecturally impossible. Re-validated every 5 minutes.

Azure advantages: Azure AD (single sign-on potential), Key Vault, cyberattack protection, Microsoft Defender, geo-redundant backups, SOC 2 Type II infrastructure, network security groups, private endpoints, industry-standard encryption for stored data.

12.9 The 14 Security Layers — Explained

What "14 Security Layers" means: The GMS implements 14 distinct, independently functioning security controls — each protecting the system at a different level. These layers work together so that even if one layer is bypassed, the remaining layers continue to protect client data:
# Security LayerWhat It Does
1Secure Password EncryptionPasswords stored as irreversible encrypted values — cannot be read even with database access
2Progressive Brute-Force LockoutEscalating cool-off periods (5→15→30→60 min) after failed login attempts
3Two-Factor Authentication (2FA)Time-based verification code as a second login step via authenticator apps (Microsoft, Google, Authy)
4Session FingerprintingSecure binding of session to IP address and device — detects hijacking
5Session Timeouts (Idle + Absolute)30-min idle timeout + 8-hour max session — prevents unattended access
6Form Validation ProtectionUnique validation tokens on all forms — prevents unauthorized form submissions
7Secure Database Queries100% secure database queries using industry-standard safety protocols
8Script Injection PreventionAll user-generated content encoded before rendering — no script injection possible
9Security HeadersComprehensive browser security headers preventing unauthorized embedding and cross-site attacks
10Role-Based Access Control (RBAC)Four-tier hierarchy enforced across every screen and integration point
11Multi-Tenant Data IsolationEvery database query filtered by organization — cross-tenant access impossible
12IP AllowlistingOrganization-level network restrictions — block access from unauthorized networks
13Immutable Audit TrailEvery action logged with user, IP, timestamp, old/new values — append-only
14HTTPS EnforcementAll traffic encrypted in transit — no HTTP access permitted
Continuous Security Enhancement: Additional security features will be added based on Citrin Cooperman's specific requirements. Examples include: advanced two-factor authentication enforcement policies, single sign-on integration via Azure AD, automated threat detection alerts, enhanced password complexity rules, device trust policies, and geo-blocking. The platform's architecture is designed to support new security layers without disrupting existing functionality.

17. Development Transparency Note

Multiple AI tools were used as development assistants only — similar to code editors.
